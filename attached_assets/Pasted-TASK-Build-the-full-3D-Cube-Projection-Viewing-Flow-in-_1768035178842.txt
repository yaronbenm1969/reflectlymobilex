TASK: Build the full “3D Cube Projection Viewing Flow” in our Expo app (for app stores), using the app’s existing colors/styles (do NOT invent new colors). The cube must feel native to the app UI.

HIGH-LEVEL EXPERIENCE (what the user sees):
1) A cinematic viewing screen with a rotating 3D cube.
2) Each cube face shows the FIRST FRAME thumbnail of the clip that will play on that face.
3) When a face becomes “active”, that face’s clip begins playing (projected onto that face).
4) The cube MUST keep gentle random motion even while a clip is playing.
5) The active face’s video continues playing even as the face moves away (locked to that face).
6) When the active face rotates away past an EXIT threshold OR the clip ends, the video fades out over 1 second.
7) Then the next face that comes into view becomes active and starts playing.
8) If there are more than 6 clips: the cube keeps 6 active faces and uses a queue; when a face’s clip ends, load the next clip into that face (thumbnail first), so it can continue indefinitely.
9) Cube motion timing must be computed from each clip’s duration: most of the clip is “readable” slow movement, last 1 second is fade-out + transition to next.

SEQUENTIAL PLAYBACK LOGIC (state machine):
- The cube always displays 6 active faces (FaceState[6]).
- Face activation:
  1) When no active face: choose the face that is most front-facing AND within ENTER_THRESHOLD (angle).
  2) Set activeFaceIndex to that face and start projecting its clip.
- While active:
  - Keep playing the same face clip even as cube rotates (locked to that face).
  - Do NOT switch just because another face becomes more front-facing.
- Stop conditions:
  - If active face goes beyond EXIT_THRESHOLD (back-facing / not visible enough) -> start 1s fade-out, then stop and clear activeFaceIndex.
  - If clip ends (didJustFinish) -> start 1s fade-out; after fade-out, replace that face’s clip from queue (if available), generate thumb, update face poster; then clear activeFaceIndex so next face can become active.
- Hysteresis:
  - ENTER_THRESHOLD < EXIT_THRESHOLD to prevent flicker.
  - Example: ENTER=20°, EXIT=60° (tune as needed).
- Crossfade:
  - Fade-out 1s then next face fade-in 0.25–0.35s.

CUBE MOTION (must be random yet gentle):
- Cube must always move:
  - Base slow auto-rotation.
  - Add subtle random wobble using smooth noise or sin waves with random phases.
  - Every 3–6 seconds pick a new small random target rotation delta; smoothly interpolate (slerp/lerp).
- Movement must be “organic” (no sharp changes).
- During playback:
  - Motion is computed from clip duration:
    Let D = durationMillis.
    Phase1 (0 → D-1000ms): keep cube mostly readable (very slow drift).
    Phase2 (D-1000ms → D): slightly increase rotation to transition + fade-out.
  - If duration is unknown: default D=8000ms until known.
